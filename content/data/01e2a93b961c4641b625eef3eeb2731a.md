# Requirements vs. Problems

In the context of these documents, [](../Requirements%20Old%20a63abb28f18e4222a758ec373413c931.md) and [Problems](../Problems%20dc5a72f9a68049c1a0424d89cf004410.md) could be handled in the same method but are separated into two processes with the following distinctions:

| **Requirements** | **Problems** |
| --- | --- |
| Can easily be broken by later design changes. | Once resolved, are not likely to resurface. |
| Can and usually should have multiple solutions. | Usually only needs a single solution. |
