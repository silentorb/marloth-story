# The problem

A key part of managing Marloth is finely optimizing the high-level view.

As long as a story has a foundation that is solid and broad, that story can hold a practically infinite variety of elements.

The high-level [[01KWN86X6NJZMP5ZESZTNDXY7W]] has a notion of priority which helps weight features.

However, I've occasionally found a need for a similar yet different dimension to further organize features.

Some features are more natural than others.

Conversely, some features are easier to loose sight of and miss.

That dimension is independent of importance.

The features that are important yet automatic end up becoming noise.

I kind of want to filter out some of the "automatic" features from daily views.

# Humor

For example, humor is an integral part of the Marloth recipe, yet I never once set out to write a humorous Marloth story.

For me, humor just sort of happens as I'm going.

> 💡While I don't set out to write humor, when I do stumble on a story element I find humorous, I *will* run with it and consciously nurture and reuse it.

While I don't consciously try to add humor, I've retroactively seen how much value humor adds to Marloth.

If something ever changed where the humor wasn't automatically entering the writing, I would want to remedy that.

In that respect, I see humor more as a requirement to be checked during later validation passes, not a feature I need to keep my eyes on while writing.
